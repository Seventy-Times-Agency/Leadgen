# Leadgen Bot — План проекта

## 1. Идея и целевой сценарий

**Продукт:** Telegram-бот для автоматического сбора B2B-лидов под услуги (изначально — таргетологи, SMM, агентства).

**Сценарий пользователя:**
1. Клиент (например, таргетолог) запускает бота.
2. Указывает нишу (например: «стоматологии», «барбершопы», «ресторанные франшизы») и регион (например: «Москва», «СПб», «Казахстан»).
3. Бот параллельно ищет компании по всем доступным источникам.
4. Возвращает пользователю список лидов: название, сайт, телефон, e-mail, соцсети, краткое описание, оценка качества лида.
5. Результат доступен в Telegram и в виде файла (Excel/CSV), опционально — выгрузка в CRM.

**Ключевая ценность:** один запрос → структурированная база потенциальных клиентов за минуты вместо часов ручного поиска.

## 3. Технологический стек (рекомендуемый)

| Слой | Технология | Почему |
|------|------------|--------|
| **Бот** | Python + [aiogram 3](https://aiogram.dev/) | Современный асинхронный фреймворк, FSM из коробки, активно развивается |
| **API (опц.)** | FastAPI | Если понадобится веб-админка/дашборд |
| **База данных** | PostgreSQL 15+ | Структурированные данные, JSONB для гибких полей, полнотекстовый поиск |
| **Очередь задач** | Celery + Redis (или arq) | Скрапинг долгий → нужен фон; Celery — стандарт, arq — легче и нативно async |
| **Кэш / брокер** | Redis 7 | Быстрый кэш результатов + брокер для очереди |
| **Скрапинг (статика)** | httpx + selectolax/BeautifulSoup | Быстрый async HTTP-клиент |
| **Скрапинг (JS)** | Playwright | Для сайтов с JS-рендером (Instagram, сложные лендинги) |
| **Поиск в сети** | SerpAPI / Serper.dev / Google Custom Search | Легальный доступ к поисковой выдаче |
| **Google Places** | Google Places API (Places API New) | Легальный источник данных о компаниях |
| **2ГИС** | Открытый API 2ГИС Catalog | Легальный, хорошо покрывает СНГ |
| **VK** | Официальный VK API | Бесплатный, много данных о сообществах |
| **Валидация email** | Python `email-validator` + DNS MX check | Базовая; потом можно добавить hunter.io/snov.io |
| **Валидация телефонов** | `phonenumbers` | Нормализация + проверка формата |
| **ORM** | SQLAlchemy 2.0 (async) + Alembic | Стандарт для Python + миграции |
| **Контейнеризация** | Docker + docker-compose | Простой деплой, воспроизводимая среда |
| **Логирование** | structlog + Sentry | Структурированные логи + отслеживание ошибок |
| **Тесты** | pytest + pytest-asyncio | Стандарт |
| **Линтеры** | ruff + mypy | Быстрый линтер + типизация |

**Почему именно Python:** лучшие библиотеки для скрапинга, зрелые обёртки под Telegram, хорошая async-экосистема, можно позже добавить ML-скоринг без смены стека.

## 4. Модель данных (предварительно)

```sql
-- Пользователи бота
users (
    id BIGINT PRIMARY KEY,              -- telegram user id
    username TEXT,
    first_name TEXT,
    created_at TIMESTAMPTZ,
    plan TEXT DEFAULT 'free',           -- free/pro/enterprise
    queries_used INT DEFAULT 0,
    queries_limit INT DEFAULT 5
)

-- Поисковые запросы
search_queries (
    id UUID PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    niche TEXT NOT NULL,
    region TEXT NOT NULL,
    status TEXT,                        -- pending/running/done/failed
    created_at TIMESTAMPTZ,
    finished_at TIMESTAMPTZ,
    leads_count INT DEFAULT 0,
    params JSONB                        -- фильтры, настройки
)

-- Лиды
leads (
    id UUID PRIMARY KEY,
    query_id UUID REFERENCES search_queries(id),
    name TEXT,
    website TEXT,
    phones TEXT[],
    emails TEXT[],
    address TEXT,
    city TEXT,
    socials JSONB,                      -- {vk: ..., instagram: ..., fb: ...}
    description TEXT,
    category TEXT,
    source TEXT,                        -- 2gis/google/vk/site/etc
    raw JSONB,                          -- сырой ответ источника
    score INT,                          -- 0-100
    dedupe_key TEXT,                    -- хэш для дедупликации
    created_at TIMESTAMPTZ
)
CREATE UNIQUE INDEX ON leads (query_id, dedupe_key);
```

## 5. Фазы разработки

### 🟢 Фаза 1 — MVP (ядро бота + 2ГИС)

**Цель:** рабочий бот, который принимает запрос и возвращает список лидов из одного надёжного источника.

- [ ] Инициализация репозитория: структура каталогов, `pyproject.toml`, docker-compose, pre-commit
- [ ] Telegram-бот на aiogram с FSM: `/start`, диалог ниша → регион → запуск
- [ ] PostgreSQL + модели (users, search_queries, leads) + Alembic миграции
- [ ] Redis + очередь задач (arq для простоты)
- [ ] Коллектор: **2ГИС Catalog API** (регистрация ключа, запросы, маппинг в `leads`)
- [ ] Коллектор: **Google Places API** (параллельный источник)
- [ ] Дедупликация (по телефону/сайту/названию + геокоду)
- [ ] Экспорт результата в Excel (openpyxl)
- [ ] Отправка результата в Telegram (сначала первые 10 карточек, потом файл)
- [ ] Базовое логирование + обработка ошибок
- [ ] Docker-образы, docker-compose, README с инструкцией по запуску

**Что уже можно показать пользователю:** вводит «кофейни Москва» → через 30 сек получает Excel с 100+ контактами.

## 6. Юридические и этические риски ⚠️

| Риск | Митигация |
|------|-----------|
| Персональные данные (152-ФЗ, GDPR) | Собирать только **публичные бизнес-контакты**, явное согласие в оферте, возможность удаления |
| Нарушение ToS соцсетей (LinkedIn, Instagram) | Использовать официальные API где возможно; предупредить пользователя; не хранить закрытые данные |
| robots.txt и rate-limit сайтов | Уважать robots.txt, ставить `User-Agent`, лимиты запросов, exponential backoff |
| Спам-рассылки по собранным лидам | **НЕ** встраивать массовую рассылку в MVP; это отдельный юридический вопрос |
| Использование данных для массовой атаки | В оферте запретить; логировать запросы |

**Важно:** LinkedIn/Instagram парсинг — всегда серая зона. Рекомендую стартовать с 2ГИС + Google Places + открытые сайты + VK API — это легально и покрывает 80% сценариев.
